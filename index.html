Areas de GIT

Working Directory

Es el area de trabajo, donde rara vez utilizaremos la linea de comandos y especialmente donde estaremos realizando las modificaciones de nuestro proyecto, como la creacion de archivos, la modificacion de los mismos e incluso su eliminacion.

  • dir || ls Muestra el contenido de la carpeta donde nos encontramos actualmente
  • cd <path> Cambia el directorio actual a la ruta especificada
  • mkdir <name> Crea una carpeta con el nombre indicado
  • git init Inicializa un repositorio de Git en el directorio actual
  • rm -rf .git Elimina el repositorio de git, el parametr -rf indica que debe eliminar la carpeta de manera forzada

Staging Area (INDEX)

Es el area de control de cambios, donde se realizan las Capturas de pantalla (Snapshot) del codigo. Todo contenido que no sea agregado a esta area, sera ignorado por GIT y por lo tanto no podra recurperarse en caso de perdida. Una vez agregado, git esta verificando constantemente si se realizaron modificaciones en el mismo.

  • git status muestra el estado de los archivos comparados con la ultima confirmacion de cambios. Los valores que puede devolver son los siguientes:
    • untracked el archivo aun no fue agregado al INDEX y por lo tanto no se le esta haciendo seguimiento.
    • tracked los cambios fueron agregados al INDEX y estan listos para ser confirmados.
    • modified el archivo al que se le esta haciendo seguimiento presenta diferencias con respecto a la ultima modificacion.
    • commited los cambios registrados fueron almacenados en el repositorio mediante una confirmacion.
  • git add <file> agregamos los archivos seleccionados al INDEX para realizarles seguimiento.
  • git blame <file> identifica a los autores de las modificaciones de un archivo en particular, siempre y cuando hayan sido confirmadas.
  • git rm --cached <file> quita del INDEX los cambios almacenados de un archivo en particular, pero mantiene el mismo en el Working Directory
  • git restore --staged <file> reestablece los cambios del INDEX de un archivo en particular, respecto de la ultima confirmacion.

Repository

Area donde se almacenan la confirmaciones de cambios en formato BLOB(Binary Large Object File), las mismas apuntan a otras confirmaciones a traves de referencias. Gracias a esto es posible navegar dentro de la historia de nuestro proyecto, para ver el estado del mismo durante las diferentes modificaciones.

  • git commit abre el editor de codigo predeterminado para generar un archivo de confirmacion.
  • git checkout <ref> permite volver a un punto anterior dentro del proyecto segun lo indicado.
  • git diff muestra las diferencias entre el Working Directory y el ultimo commit.
  • git log muestra el historial de confirmaciones en forma de arbol, con los numeros de HASH pertinentes.
  • git remote add <alias> <url> agrega una direccion como repositorio remoto.
  • git push <remote> <branch> sube los cambios realizados al repositorio remoto.
